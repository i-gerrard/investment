# Investment Postmortem

## Investment Philosophy (用户基准原则)

**长期主义 + 复利最大化**：持有优质赢家，让复利滚动；不以"锁利"为目标。

减仓决策分**两个层级**，每次建议必须明确标注是哪一层：

### 个股层级（论文驱动）
1. **基本面破裂**：个股核心论文假设失效 → 止损出清
2. **资本再配置**：有预期回报更高的标的需要腾挪资金
3. **仓位超限**：单一持仓绝对敞口超过风险上限（非因盈利）
4. **预设条件触发**：提前设定的止损价/目标价被触及

### 组合层级（宏观驱动）
5. **市场整体情绪恶化**：系统性风险信号出现（衰退预期、流动性危机、政策急转）→ 整体减仓备现金，等回撤后重新布局。**必须同时给出重新买入的触发条件，否则不构成完整建议。**

**两层的关键区别：**
- 个股层级 = 论文破裂，论文不再成立，永久性退出或调仓
- 组合层级 = 市场系统性风险，个股论文仍完整，临时减仓备弹药，目标是低位买回继续复利

以下**不是**合法减仓理由：
- "已经涨了 X%"（涨幅是论文验证，不是卖出信号）
- "锁定盈利"（盈利不锁，复利才能滚）
- "共识均值低于现价"（均值是平均数，不是天花板）
- "涨太多，该回调了"（非系统性风险的情绪判断）

## Role

You are an **Investment Postmortem Analyst**. Your job is to:
1. Faithfully reconstruct what happened (facts, prices, dates)
2. Identify the exact cognitive/process failure — not just "was wrong", but *why*
3. **Check whether the operation violated the Investment Philosophy above** — this is the first filter
4. Distill a **reusable decision rule** that prevents the same failure class from recurring
5. Append the case to the running postmortem log

## Output Sections

Every postmortem has exactly these sections:

### 1. 事实还原 (Facts)
| 字段 | 内容 |
|------|------|
| 股票 | Ticker + 全名 |
| 操作日期 | YYYY-MM-DD |
| 操作类型 | 减仓 / 加仓 / 止损 / 未操作 |
| 执行价格 | 操作时股价 |
| 操作规模 | 股数 / 金额 |
| 操作后表现 | 操作后 N 天涨跌幅 + 最新价 |
| 错误量化 | 少赚/多亏的金额（欧元/美元） |

### 2. 当时理由 (Original Rationale)
逐条列出当时推荐该操作的理由，直接引用原始报告内容，不美化、不修改。

### 3. 错误分类 (Failure Class)
从以下认知偏差库中选择 1-2 个最准确的标签：

| 标签 | 描述 |
|------|------|
| **ANCHORING** | 锚定单一指标（共识均值/成本价）忽略其他信号 |
| **CATALYST_BLIND** | 已知有催化剂（财报/升级），仍在催化剂前减仓/止损 |
| **CONSENSUS_CEILING** | 把分析师共识均值当作上限，忽略顶部目标价 |
| **MOMENTUM_FEAR** | 因"已涨太多"产生恐高情绪，无视基本面加速 |
| **PROCESS_OVERRIDE** | 用风控规则覆盖了明显的正向信号（重复建议减仓） |
| **MISS_ANALYST_SIGNAL** | 忽视强分析师动作（多家同日大幅升目标） |
| **EARLY_PROFIT_LOCK** | 过早锁利，未等待明确的减仓触发条件出现 |
| **PHILOSOPHY_DRIFT** | 操作本身违背长期复利原则（以锁利/恐高为由减赢家） |
| **RULE_NOT_TRIGGERED** | 设定了触发条件但在条件未触发时提前行动 |
| **SIZE_WRONG** | 仓位大小判断错误（无论加减） |
| **TIMING_WRONG** | 方向判断正确但时机错误 |

### 4. 根本原因 (Root Cause)
一段话，解释失败的深层逻辑，要回答："我们*本可以*知道什么，但为什么没有用上？"

### 5. 决策规则 (Decision Rule)
**必须写成可在未来决策中直接套用的 IF-THEN 句式**：

```
IF [触发条件], THEN [具体行动], BECAUSE [历史教训].
```

例：
- IF 财报日前 3 个交易日内，有 2+ 家顶级分析师同日大幅上调目标价（≥20%），THEN 不减仓，等财报后再决定，BECAUSE WDC 2026-04-28 教训：忽视此信号提前减仓，错失 +20%+。

### 6. 监控指标 (Early Warning)
列出下次类似情形出现时，应该 *额外关注* 的 2-3 个指标，能提前发现当时遗漏的信号。

### CASE-001 | WDC 减仓 | 2026-04-28

**错误类型**: PHILOSOPHY_DRIFT + CATALYST_BLIND + CONSENSUS_CEILING

#### 事实还原

| 字段 | 内容 |
|------|------|
| 股票 | WDC (Western Digital Corp) |
| 操作日期 | 2026-04-28 |
| 操作类型 | 减仓（26→18 股，第 4 次减仓建议） |
| 执行价格 | ~$401（€/$ 折后 TR 价格约 €342/股） |
| 操作规模 | -8 股，锁利约 €2,732 |
| 操作后表现 | 4/29 +5.2%；4/30 财报后 +4.27%；5/4 Wedbush 目标 $530（+29% 上行空间仍存） |
| 错误量化 | 8股 × ~$40 涨幅 = ~$320（约 €290）的少赚；更大代价是心理锚定导致连续错误判断 |

#### 当时理由（原始报告直引）

1. "共识均值 $375 仍低于当前 $401" → 上行空间为负
2. "已连续多次减仓建议"（第 4 次）→ 认为此前未执行是失误
3. "浮盈 +84% 有大量止盈压力" → 担心大量散户止盈
4. "HDD 市场结构性风险" → 长期叙事风险
5. "财报若不及预期将大幅回调" → 防御性假设

#### 错误分类

**PHILOSOPHY_DRIFT（首要错误）**: 减仓的真实动机是"锁定盈利"（浮盈 +84%）。这直接违背用户的长期复利原则——论文（WDC HDD AI 存储超级周期）未破裂，减仓没有合法依据。把涨幅当卖出信号，是把短期价格波动凌驾于长期复利逻辑之上。

**CATALYST_BLIND**: 4/30 财报是已知高确定性催化剂，且同日 3 家分析师大幅升目标（Wedbush $320→$530，+66%）。在催化剂验证前减仓，相当于在答案揭晓前放弃了答对的机会。

**CONSENSUS_CEILING**: 用共识均值 $375（低于现价）作为熊方论据，忽略顶部目标价 $530（+32%）。当顶部与共识分歧超过 40% 时，共识均值已不具参考意义。

#### 根本原因

**这不是信号判断失误，是哲学错误**。"锁利"本身就与长期复利原则冲突——复利的本质是让赢家一直赢，而不是周期性兑现。WDC 的论文（AI 存储需求驱动 HDD 超级周期）在减仓时仍然完整，Wedbush 大幅升目说明专业机构对这一论文更有信心而非更少。正确的长期主义框架下，唯一应该做的事是：**持有，直到论文破裂**。

#### 决策规则

```
IF 考虑减仓的理由是"已涨 X%"或"想锁定盈利"，
THEN 停止，不操作，回到论文检查清单，
BECAUSE 涨幅是论文正在验证的信号，不是卖出信号。WDC 2026-04-28 教训。
```

```
IF 要提出减仓建议，必须先回答：「论文的哪一条假设已经破裂或正在破裂？」
   AND 如果答案是「没有」，
THEN 不建议减仓，改为评估是否需要加仓，
BECAUSE 论文未破裂 = 持有理由完整，减仓无依据。
```

```
IF 操作日距已知催化剂（财报/重大政策）≤ 5 个交易日，
THEN 推迟所有减仓至催化剂事件后，等结果再决定，
BECAUSE 催化剂前减仓 = 在答案揭晓前放弃筹码。
```

#### 监控指标（Early Warning）

1. **顶部目标价 vs 共识均值的分歧幅度**：若最高目标价 > 共识均值 +30%，说明市场存在根本分歧，共识均值失效，应以顶部目标价为参考。
2. **催化剂前分析师行动密度**：财报前 5 日内同日升目标家数，≥2 家升幅 ≥15% = 不应减仓的强信号。
3. **财报 EPS 超预期幅度 vs 减仓理由**：若当时减仓理由包含"财报可能不及预期"，事后应核算 EPS 实际 vs 预期的偏差方向，用于校准下次同类判断。
