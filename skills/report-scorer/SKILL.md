# Report Scorer

## 触发条件

用户说"给报告打分"、"report 打分"、"评分"、"质检"、"/report-scorer" 时触发。

默认对当天报告打分：`~/Desktop/claude/{today}/report-{today}.html`

用户可指定路径或日期，如：`/report-scorer 2026-05-08`

---

## 评分维度（满分 100）

| 维度 | 权重 | 评分要点 |
|------|------|---------|
| **① 数据完整性** | 20分 | 所有持仓有价格、均价、盈亏；现金数据准确；汇率/指数数据存在 |
| **② 个股结论质量** | 30分 | 结论有具体论据；重仓股有深度分析；结论与数据自洽 |
| **③ 操作推荐质量** | 30分 | 止损价有技术/逻辑依据；亏损仓位有止损；建议可直接执行；优先级分层 |
| **④ 风险识别** | 20分 | 现金比例风险、集中度风险、事件风险（财报/宏观）均有覆盖 |

---

## 评分规则

### 数据完整性（满分 20）

扣分项：
- 持仓量显示"—" → -3/项（最多扣 6）
- 均价缺失 → -2/项
- 盈亏缺失 → -2/项
- 现金占比未计算 → -4
- 汇率/指数数据缺失 → -2

### 个股结论质量（满分 30）

扣分项：
- 亏损仓位结论为"等待修复"但无止损 → -8
- 大持仓（市值前5）结论不足2句，无止损 → -6/项
- 催化剂未知时，结论仍基于猜测性假设 → -5
- 结论与当日涨跌数据明显矛盾 → -5

加分项（从80分起算）：
- 正确识别当日异常波动并给出应对 → +3
- 坚守投资哲学（论文未破裂不减仓） → +3

### 操作推荐质量（满分 30）

扣分项：
- 止损价格为整数取整，无技术/逻辑支撑 → -8
- 亏损最大持仓在操作表中缺席 → -8
- 现金偏低但无具体补充方案 → -6
- 全仓 HOLD，无任何差异化分析（强势日是否有补仓机会） → -4
- 数据缺失持仓在操作表无"待核实"标注 → -4

加分项（从80分起算）：
- 止损价格明确可执行（有具体价格） → +3
- 优先级分层（必须/关注/持有） → +2

### 风险识别（满分 20）

扣分项：
- 未提示现金比例风险 → -6
- 未提示大仓位集中度风险 → -4
- 未提示已知近期事件（财报/FOMC） → -4
- 单日暴涨未提示回吐风险 → -4

## 输出格式

### 文字输出（给用户）

```
## 报告评分：结论与操作推荐

### 结论质量：XX / 100
[加分项列表]
[扣分项列表，每项说明原因和改进建议]

### 操作推荐质量：XX / 100
[加分项列表]
[扣分项列表]

### 综合评分
| 维度 | 得分 |
|------|------|
| 数据完整性 | XX |
| 个股结论质量 | XX |
| 操作推荐质量 | XX |
| 风险识别 | XX |
| **综合** | **XX / 100** |

**最优先修复：** ① ... ② ... ③ ...
```

### HTML 嵌入（写入报告文件）

在报告的 `<footer>` 前插入 `<h2>五、报告质量评分</h2>` 区块，包含：
1. 综合评分大卡片（深色渐变背景）
2. 四维度分数网格（2×2）
3. 加分项列表
4. 扣分项列表（每项含扣分值、标题、说明和建议）
5. 最优先修复提示条（黄色 catalyst 样式）

嵌入使用 HTML `<style>` 局部样式，避免污染报告整体样式。

---

## 投资哲学对齐检查（必须执行）

在评分前，先检查报告是否有以下哲学违规：

| 违规类型 | 表现 | 处理 |
|---------|------|------|
| 减仓建议无合法理由 | 仅因"涨太多"、"锁利"建议减仓 | 在结论评分中扣 -10，并标注违规 |
| 亏损持仓无止损 | 亏损仓位结论为"持有等待" | 在操作推荐中扣 -8 |
| 止损设置偏紧 | 止损在均价下方 <5% | 标注警告，不扣分但注明风险 |

---

## 执行流程

```
1. 确定报告路径（默认今日，可指定日期）
2. Read 报告 HTML
3. 按四维度评分，记录每条扣/加分理由
4. 投资哲学对齐检查
5. 汇总分数，生成文字输出给用户
6. 将评分 HTML 区块插入报告文件（在 footer 前）
```
